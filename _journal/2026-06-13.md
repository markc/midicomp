## Changes

- Flipped license headers in `midicomp.c` (usage banner + footer block),

### Build fix
- `CMakeLists.txt` failed on modern CMake (required `< 3.5`). Bumped
  `cmake_minimum_required` to 3.10 and moved it before `project()`.
- The code is pre-ANSI K&R C and won't compile under a modern default C standard,
  so pinned `CMAKE_C_STANDARD 90` with extensions on (gnu89). Build now configures
  and compiles cleanly (warnings only, no errors).

### Tests
- Added `tests/run_test.cmake` — a pure-CMake CTest driver (no shell/bash/python,
  so it runs anywhere CMake does, which suits a public C project).
- Wired `enable_testing()` + 4 `add_test()` calls into `CMakeLists.txt`. Run with
  `ctest`. All 4 pass:
  - **plain** — decode of `ex1.mid` matches `ex1-plain.txt`.
  - **verbose** — decode `-v -t` matches `ex1-verbose.txt`.
  - **roundtrip** — decode → recompile → decode is idempotent.
  - **canonical** — midicomp's own SMF output recompiles byte-identically.

### Docs
- New `CLAUDE.md` documenting build, two-path callback architecture
  (`mfread` → `my*` printers vs flex lexer → `mf_w_*` writers), grammar
  regeneration, and gotchas.
- Updated `README.md`: MIT license, gnu89/CMake 3.10 build note, test section,
  v0.0.9 changelog entry.

## Notes

- `ex1-verbose.txt` was generated with `-v -t`, not `-v` alone — it uses absolute
  `mmm:bb:ttt` timestamps. Corrected the assumption in the docs and test.
- A recompiled `ex1.mid` is NOT byte-identical to the original (148 → 180 bytes):
  the original uses MIDI running status (omitted repeated status bytes), which
  midicomp expands to explicit status. Correct lossless behaviour; byte-stability
  only holds gen-1 onward, which is what the `canonical` test asserts.
- `midicomp -c -` writes to a literal file named `-`, it does NOT stream SMF to
  stdout. Cleaned up a stray `-` file this created during testing.
- 32 pre-existing compiler warnings remain (K&R style: missing returns, implicit
  ctype declarations, char/unsigned-char signedness). Two cosmetic real bugs noted
  but left out of scope: extra `fprintf` arg at midicomp.c:965, undersized `mknote`
  buffer near line 690.

## Warning cleanup — zero-warning -Wall build

Follow-up to the morning session: cleared all 32 warnings and the two cosmetic
bugs flagged above, then enabled `-Wall` in the build to keep it that way.

### Cosmetic bugs fixed
- `midicomp.c:851` — `printf("Arb", leng)` had a stray arg with no format
  specifier; now `printf("Arb")`.
- `midicomp.c:965` — `fprintf(stderr, "Error: Garbage at end\n", s)` had an unused
  `s` arg; removed.
- Bonus real fix: `mknote`'s `buf[5]` was too small for all note-name+octave
  combinations the compiler saw as reachable (`-Wformat-overflow`). Enlarged to
  `buf[16]`.

### Warnings cleared (midicomp.c / midicomp.h — now genuinely clean)
- Added `#include <ctype.h>` (4 implicit isprint/isupper/tolower/islower).
- `metaevent`/`sysex`/`chanmessage` → `void` (returns were never used; 3 missing-
  return warnings).
- Explicit return types on `myerror`/`getbyte`/`getint`/`fileputc` (4 default-int).
- `return 0;` added to `main`.
- Forward decls for `getint`/`getbyte`/`yylex` (3 implicit-declaration).
- `data[5]` and `buffer` → `unsigned char` to match the `mf_w_*` writer signatures
  (11 pointer-signedness warnings; MIDI bytes are unsigned, behaviour identical).
- Removed unused `verbose_flag`.
- `Mf_error` → `void (*)()` to match the now-void `myerror` and the other handlers.

### Headers
- Removed the dead `static` forward-decl block from `midifile.h` (9 "declared
  static but never defined" warnings in the lexer TU).
- Added `#include <unistd.h>` + `void error();` to `t2mf.h` (implicit read/error in
  the generated lexer).

### Generated lexer
- `t2mflex.c` has 4 warnings inherent to flex output (char-subscript table lookups,
  always-emitted `input`/`yyunput`). Not fixable in source. Enabled `-Wall` on the
  target and suppressed only `-Wno-char-subscripts -Wno-unused-function` on that one
  generated file via `set_source_files_properties`. Our own code carries no
  suppressions.

Result: `cmake .. && make` builds with zero warnings; `ctest` 4/4 green.
